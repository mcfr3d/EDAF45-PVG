Installering och körning

Unzippa filen release.zip och placera mappen release på ett lämpligt ställe på datorn.

Delapplikationer

Programmet består av två stycken delapplikationer. Dels ett registreringsprogram och dels ett resultatprogram.

Registreringsprogrammet

  • Beskrivning - Namnet på programmet är registration.jar och används för att registrera tider under tävlingar. Både starttider och sluttider.

  • Körning - För att köra programmet kan man antingen öppna filhanteraren, navigera in i mappen release och dubbelklicka på filen registration.jar. Eller så kan man köra programmet via terminalen genom att navigera in i mappen release och sedan skriva: java -jar registration.jar .

  • Användning - När programmet öppnar får man först välja en fil som tiderna ska sparas till. Därefter öppnas en ruta för konfiguration av serveranslutning. Eftersom server-klient lösningen inte är fullt integrerad i denna release kan du här trycka på avbryt. När det är gjort öppnas själva programmet som man registrerar tider i. För att registrera en tid så skriver man in förarens startnummer, t.ex ”12”, i textfältet och trycker sedan på enter alternativt på knappen ”Registrera”. Förarens startnummer och tiden syns då till vänster i rutan nedan. För att registrera tiden för flera förare samtidigt kan man skriva på följande olika sätt:

    • Flera förare - t.ex ”5 8 7 11”
    • Intervall - t.ex ”1-9”
    • Klass - måste inledas med ett @-tecken t.ex ”@Seniorer” (måste skrivas exakt som klassen är skriven i namnfilen)
    • Kombination av de ovannämnda sätten - t.ex ”6 12 15-18 @Seniorer”

    Om man inte kan läsa en förares startnummer på grund av t.ex smutsig nummerplåt så kan man lämna textfältet tomt och registrera tiden ändå. Registreringen syns då till höger i rutan nedan där man har chans att i efterhand skriva in startnumret. Likaså om man skriver någon felaktig input i textfältet så hamnar registreringen till höger där man har möjlighet att korrigera felet. Resultatet av körningen av programmet är en textfil som innehåller alla de registrerade tiderna. Denna fil används sen av resultatprogrammet för att generera resultatfiler.

Resultatprogrammet

  • Beskrivning - Namnet på programmet är resultMerge.jar och används för generera filer med resultatet från en tävling.

  • Körning - För att köra programmet behöver man först skapa en konfigurationsfil som man ger som parameter till programmet. I mappen release finns filen templateconfig.json som visar exempel på hur konfigurationsfilen kan se ut och hur man fyller i den. De parametrar som man ger programmet via konfigurationsfilen är:

    • race type - Kan vara ”maraton”, ”varvlopp” eller ”etapplopp”. Default är ”maraton”.
    • group start - Tiden för eventuell masstart.
    • name file - Lokala sökvägen till namnfilen. Krävs för att kunna köra programmet.
    • stipulated time - Stipulerad tid. Krävs om race type är angett till ”varvlopp”.
    • number of etapps - Antalet etapper vid etapplopp. Krävs om race type är angett till ”etapplopp”.
    • start files - Lokala sökvägen till filen med starttider. Flera filer kan anges genom att kommaseparera dem. Om race type är angett som ”etapplopp” anges start files istället som objekt innehållandes:
      • etapp - Numret på etappen.
      • file - Lokala sökvägen till filen med starttider för etappen.
    • finish files - Lokala sökvägen till filen med sluttider. Flera filer kan anges genom att kommaseparera dem. Om race type är angett som ”etapplopp” anges finish files istället som objekt innehållandes:
      • etapp - Numret på etappen.
      • file - Lokala sökvägen till filen med sluttider för etappen.
    • etapper - Info om etapperna. Krävs om race type är angett till ”etapplopp” och måste anges för etapperna i nummerordning. Anges som objekt innehållandes:
      • minimum time - Minimala tiden för etapper.
      • multiplier - Vikten av etappen. Tiden multipliceras med multiplier.

I mappen config finns det tre exempel på configurationsfiler, config_maraton.json, config_varv.json, config_etapp.json

När konfigurationsfilen är skapad och korrekt ifylld kan man förslagsvis döpa den till config.json och placera den i mappen release. Därefter kan man köra programmet via terminalen genom att navigera in i mappen release och sedan skriva: java -jar resultMerge.jar -c config.json . Resultatet av körningen blir filerna:

  • output_unsorted.txt - .txt fil innehållandes det osorterade tävlingsresultatet.
  • output.txt - .txt fil innehållandes det sorterade tävlingsresultatet.
  • output.html - .html fil innehållandes det sorterade tävlingsresultatet.
